1. Was ist die OMG?
 
 

Eine nicht profit-orientierte Gruppe mit über 800 Mitgliedern.
Die OMG erarbeitet Spezifikationen zur Verteilung und Kommunikationen zwischen Objekten.

bekannteste Spezifikationen: CORBA, UML, MDA

 

2. Was ist der ORB?
 
  Object Request Broker ist Laufzeitsystem; transportiert Aufrufe zwischen den CORBA Objekten  

3. Was ist IIOP?
 
  Protokoll zur Kommunikation zwischen ORBs  

4. Was ist IDL?
 
  Interface Definition Language ist Sprache zur Beschreibung von Schnittstellen in OMA (Standard für offenes verteiltes System)  

5. Nennen Sie drei Unterschiede zwischen Java RMI und CORBA.
 
 

- RMI ist sprachgebundendene Plattform von Sun, CORBA ist dagegen sprach- und herstellerunabhängig
- bei CORA sind Mappings auf die jeweilige Sprache notwendig, bei RMI nicht
- bei CORA sind nur Referenzen auf Objekte möglich, bei RMI können Objekte (durch nachladen) als Argumente verwendet werden
- RMI hat ein Laufzeitsystem, in CORBA sind unterschiedliche Laufzeitsysteme (ORBs) integriert

7. Welche ganzzahligen Zahlentypen sind in IDL vorgesehen?
 
  short, long, long long  

8. Welche Fliesskommatypen gibt es in IDL?
 
  float, double, long double, fixed  

9. Was kann in einem Modul in IDL vorkommen?
 
  Modul ist Namensraum für zusammengehörige Schnittstellen (vergleichbar: package in java) und kann Schnittstellen und Typdefinitionen enthalten.  

10. Was kann in einer Schnittstelle in IDL vorkommen?
 
  Konstanten, Attribute, Typdefinitionen , Operationen, Ausnahmedefinitionen  

11. Welche Arten von Parameterüberhabe kennt CORBA?
 
 
  • in: Parameter wird vom Client zum Server geschickt (by value)
  • out: Parameter wird vom Server zum Client geschickt (Ergebnis)
  • inout: Parameter wird in beide Richtungen geschickt (änderbar)
